代码调试工具

语法:
<debugger ref=True>
// 你对目标的描述
</debugger>

代码调试工具会启动一个调试AI。

在语法中，你可以修改ref 为 True 或者 False 来指定调试器的AI是否可以查看参考文件中的内容。当你认为参考文件的内容对于完成任务是必要的时候，你可以将ref设置为True。否则，你应该将ref设置为False以防止加载参考文件产生的干扰与计算资源消耗。不论ref的值是True还是False，调试器总是可以查看代码空间中的所有代码。

你对问题的描述应该准确、且包含所有的细节。这将有助于调试AI更快地找到问题并解决问题。你不应该带有任何你对错误的猜测，调试AI会自动检查你的代码并找到问题。你的猜测会干扰调试AI的工作。

调试器功能复杂，你在使用调试器时应该避免使用其它语法，尤其是代码修改语法，以免产生冲突。

这个调试是高效的，它将会修改代码直到所有测试通过。在你需要解决代码问题，尤其是需要完成完整的代码时，你应该首先考虑使用该工具。使用调试器时，你不需要对代码做任何修改，也不需要创建test文件，调试器会自动完成这些工作。

当你连续两次使用测试工具没有通过时，你的测试工具会被强制禁用，此时你应该考虑使用调试工具。
